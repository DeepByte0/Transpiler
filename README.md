# Transpiler

> Лёгкий транспайлер, который преобразует арифметические выражения в код **GAS (GNU Assembler)** для архитектуры **x86-64** в синтаксисе **AT&T**.

Написан на C++ с нуля, чтобы разобраться, как устроен компилятор изнутри — от разбора исходного текста до генерации низкоуровневого кода.

## Описание

Программа принимает на вход арифметическое выражение (числа, операторы `+`, `-`, `*`, `/` и скобки), учитывает приоритет операций и выдаёт эквивалентный код на ассемблере GAS в синтаксисе AT&T. Это учебный проект: цель была не в том, чтобы сделать промышленный компилятор, а в том, чтобы вручную реализовать каждый классический этап его работы и увидеть, как высокоуровневое выражение превращается в машинные инструкции.

## Как это работает

Транспайлер обрабатывает входное выражение в четыре этапа:

1. **Лексический анализ** — входная строка разбивается на токены: числа, операторы и скобки.
2. **Синтаксический анализ** — с помощью [алгоритма сортировочной станции (Shunting Yard)](https://ru.wikipedia.org/wiki/Алгоритм_сортировочной_станции) строится абстрактное синтаксическое дерево (AST) с корректным приоритетом и ассоциативностью операций.
3. **Представление в виде AST** — дерево состоит из двух типов узлов: `Num` (лист, хранит число) и `Op` (внутренний узел, хранит оператор и его операнды).
4. **Генерация кода** — дерево обходится рекурсивно, и каждый узел порождает соответствующие инструкции GAS в синтаксисе AT&T.

## Сборка и запуск

Проект — обычное C++-приложение без внешних зависимостей.

**Visual Studio (Windows):**

1. Откройте `Transpiler.slnx` в Visual Studio 2026.
2. Соберите решение (`Ctrl + Shift + B`).
3. Запустите полученный исполняемый файл и введите арифметическое выражение.

**Любой компилятор C++ (g++ / clang):**

```bash
# из директории с исходником
g++ -std=c++17 -O2 main.cpp -o transpiler
./transpiler
```

Идеи для дальнейшего развития проекта:

## Зачем этот проект

Это курсовая работа, но она охватывает базовые идеи, лежащие в основе компиляторов и низкоуровневой генерации кода: токенизацию, синтаксический разбор с явной обработкой приоритета, построение и обход AST, а также понимание того, как конструкции исходного языка превращаются в ассемблер. Проект отражает мой интерес к системному и низкоуровневому программированию.
